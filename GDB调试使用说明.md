# GDB调试功能使用说明

## 概述

本项目集成了增强的GDB远程调试服务器，支持ARM架构的固件调试。GDB服务器默认监听端口6688。

## 启动GDB调试

### 1. 启动模拟器
运行编译后的main.exe，模拟器会自动启动GDB服务器。

### 2. 连接GDB客户端
使用ARM GDB客户端连接到模拟器：

```bash
arm-none-eabi-gdb
(gdb) target remote localhost:6688
```

## 支持的GDB命令

### 基本命令

#### `?` - 查询停止原因
查询目标停止的原因，返回信号编号。

**示例：**
```
(gdb) ?
```

**返回信号：**
- `S05` - TRAP信号（单步执行停止）
- `S07` - 断点命中
- `S0A` - 观察点触发

#### `g` - 读取所有寄存器
读取所有ARM寄存器的值（R0-R15, CPSR）。

**示例：**
```
(gdb) g
```

#### `m addr,length` - 读取内存
从指定地址读取内存内容。

**参数：**
- `addr` - 内存地址（十六进制）
- `length` - 读取长度（字节）

**示例：**
```
(gdb) m 0x20000000,16
```

#### `M addr,1:value` - 写入内存
向指定地址写入一个字节。

**参数：**
- `addr` - 内存地址（十六进制）
- `value` - 写入的值（十六进制）

**示例：**
```
(gdb) M 0x20000000,1:0xFF
```

#### `P reg=value` - 写入寄存器
修改指定寄存器的值。

**参数：**
- `reg` - 寄存器编号（0-16）
- `value` - 寄存器值（十六进制）

**寄存器映射：**
- 0-15: R0-R15
- 16: CPSR

**示例：**
```
(gdb) P 0=12345678
```

### 执行控制命令

#### `s` - 单步执行
执行一条指令后停止。

**示例：**
```
(gdb) s
```

#### `c` - 继续执行
继续执行直到遇到断点或观察点。

**示例：**
```
(gdb) c
```

### 断点命令

#### `Z0,addr` - 设置软件断点
在指定地址设置软件断点。

**参数：**
- `addr` - 断点地址（十六进制）

**示例：**
```
(gdb) Z0,0x1000
```

#### `z0,addr` - 删除软件断点
删除指定地址的软件断点。

**参数：**
- `addr` - 断点地址（十六进制）

**示例：**
```
(gdb) z0,0x1000
```

#### `Z1,addr` - 设置硬件断点
在指定地址设置硬件断点。

**参数：**
- `addr` - 断点地址（十六进制）

**示例：**
```
(gdb) Z1,0x2000
```

#### `z1,addr` - 删除硬件断点
删除指定地址的硬件断点。

**参数：**
- `addr` - 断点地址（十六进制）

**示例：**
```
(gdb) z1,0x2000
```

#### `z3,addr` - 删除读观察点

### 查询命令

#### `qSupported` - 查询支持的特性
查询GDB服务器支持的特性。

**示例：**
```
(gdb) qSupported
```

**返回：**
```
PacketSize=4096;QStartNoAckMode+;swbreak+;hwbreak+;vContSupported+
```

#### `vCont?` - 查询支持的继续命令
查询支持的继续执行命令。

**示例：**
```
(gdb) vCont?
```

**返回：**
```
vCont;c;C;s;S;t
```

#### `vCont;action` - 继续执行
使用vCont协议继续执行。

**参数：**
- `action` - 执行动作（c=继续, s=单步）

**示例：**
```
(gdb) vCont;c
(gdb) vCont;s
```

#### `qfThreadInfo` / `qsThreadInfo` - 查询线程信息
查询目标系统的线程信息。

**示例：**
```
(gdb) qfThreadInfo
(gdb) qsThreadInfo
```

#### `qThreadExtraInfo,id` - 查询线程额外信息
查询指定线程的额外信息。

**示例：**
```
(gdb) qThreadExtraInfo,1
```

#### `Hc` / `Hg` - 设置线程操作
设置当前操作的线程。

**示例：**
```
(gdb) Hc1
(gdb) Hg1
```

#### `T` - 检查线程存活
检查指定线程是否存活。

**示例：**
```
(gdb) T1
```

### 监控命令

#### `qRcmd,command` - 执行监控命令
执行自定义监控命令。

**支持的命令：**
- `backtrace` 或 `bt` - 显示调用栈

**示例：**
```
(gdb) qRcmd,6261636b7472616365
```

（注：命令需要转换为十六进制编码）

## 高级功能

### 1. 条件断点
支持基于寄存器值的条件断点。

**支持的寄存器：**
- R0, R1, R2, R3

**支持的操作符：**
- `==` - 等于
- `!=` - 不等于
- `>` - 大于
- `<` - 小于

**示例：**
```
(gdb) break *0x1000 if r0==0x1234
(gdb) break *0x2000 if r1!=0
(gdb) break *0x3000 if r2>0x100
```

### 2. 调用栈回溯
使用监控命令查看调用栈。

**示例：**
```
(gdb) monitor backtrace
(gdb) monitor bt
```

**输出示例：**
```
#0  0x00001000 in ?? ()
#1  0x00002000 in ?? ()
#2  0x00003000 in ?? ()
...
```

### 3. 观察点
监控内存地址的读写操作。

**写观察点：**
```
(gdb) watch *0x20000000
```

**读观察点：**
```
(gdb) rwatch *0x20000000
```

**访问观察点：**
```
(gdb) awatch *0x20000000
```

### 4. 硬件断点
使用硬件断点功能，不会修改目标内存。

**示例：**
```
(gdb) hbreak *0x1000
```

## 停止原因详解

GDB服务器会返回不同的停止原因信号：

| 信号 | 含义 | 触发条件 |
|------|------|----------|
| S05  | TRAP | 单步执行停止 |
| S07  | 断点命中 | 软件断点或硬件断点触发 |
| S0A  | 观察点触发 | 读/写观察点触发 |

## 使用技巧

### 1. 快速设置断点
在GDB中使用标准断点命令：

```
(gdb) break *0x1000
(gdb) break main
```

### 2. 查看反汇编
```
(gdb) disassemble 0x1000, +20
```

### 3. 查看内存
```
(gdb) x/10x 0x20000000
(gdb) x/20i 0x1000
```

### 4. 保存和加载断点
```
(gdb) save breakpoints my_breakpoints.txt
(gdb) source my_breakpoints.txt
```

## 限制和注意事项

1. **断点数量限制：** 最多支持32个断点
2. **观察点数量限制：** 最多支持16个观察点
3. **条件断点：** 仅支持R0-R3寄存器的简单条件
4. **调用栈深度：** 最多显示32层调用栈
5. **内存访问：** 观察点仅在有效内存区域内工作

## 常见问题

### Q: GDB连接失败
A: 确保模拟器正在运行，并且防火墙没有阻止6688端口。

### Q: 断点不生效
A: 检查地址是否在可执行内存区域内，确保地址正确。

### Q: 观察点不触发
A: 确保观察点地址在有效内存范围内，并且有实际的读写操作。

### Q: 单步执行不工作
A: 检查目标是否在Thumb模式下，确保指令对齐正确。

## 技术细节

### GDB协议实现
- 使用标准GDB远程串行协议
- 支持数据包校验和验证
- 支持中断信号（0x03）处理

### ARM支持
- 支持ARM和Thumb模式
- 支持所有ARM寄存器（R0-R15, CPSR）
- 支持条件断点

### 内存管理
- 使用Unicorn Engine的内存访问接口
- 支持内存读写观察点
- 支持内存断点
